ONBOARDING A NEW CLIENT (Step by Step)
---------------------------------------
Step 1 — Client registers
   - Send them: qrwegn.com/register
   - They create their account and choose a plan
   - They appear automatically on your /admin page with NOT STARTED status

Step 2 — Set up their tables
   - Open their Deploy Kit on /admin, click Dashboard
   - Go to the Tables tab and add their tables (e.g. Table 1 to Table 10)
   - Checklist auto-checks "Tables set up"

Step 3 — Load their menu
   - Go to the Menu tab in their dashboard
   - Add categories and menu items with prices
   - Checklist auto-checks "Menu loaded"

Step 4 — Print and deliver their QR code
   - On /admin, click the QR code image to download it
   - Print it and place it on each table
   - Click "QR code printed" in the Communication Log

Step 5 — Train staff
   - Give staff their login details:
     URL: qrwegn.com/staff-login
     Restaurant ID: their slug (e.g. red-sea)
     PIN: their staff PIN (default 2026, update on /admin if needed)
   - Show them the kitchen view and how to update order status
   - Click "Staff trained" in the Communication Log

Step 6 — Go live
   - Customer scan page is ready: qrwegn.com/scan/{slug}
   - Click "Went live" in the Communication Log
   - Status changes to LIVE automatically after first order received

TROUBLESHOOTING
---------------

Problem: Scan page shows blank or "restaurant not found"
Fix: Check the slug in the URL is correct. Check Supabase businesses table for the correct slug.

Problem: Staff login shows "Restaurant not found"
Fix: Confirm the slug matches exactly what is in the businesses table (lowercase, hyphens only).

Problem: Orders not appearing in kitchen view
Fix: Hard refresh (Ctrl+Shift+R). Kitchen polls every 15 seconds. Check that the order was placed successfully on the scan page.

Problem: Cancel button shows schema error
Fix: The cancel_reason column may be missing. Run in Supabase SQL editor:
ALTER TABLE orders ADD COLUMN IF NOT EXISTS cancel_reason text;
ALTER TABLE orders ADD COLUMN IF NOT EXISTS cancelled_at timestamptz;

Problem: Financials showing $0.00
Fix: Check that orders have status other than 'cancelled'. Financials only count completed orders.

Problem: Mobile dashboard shows blank page
Fix: This was caused by the Notification API not being available on iOS Safari. Should be fixed. If it recurs, check for any direct use of "Notification" without typeof check.

Problem: Vercel deployment failing
Fix: Check Vercel dashboard at vercel.com for the error. Most common causes are unused imports or missing columns. Paste the error into Claude Code and ask it to fix.

Problem: Staff PIN not working
Fix: Go to /admin, find the business, update the PIN in the Staff PIN field and click Update.
